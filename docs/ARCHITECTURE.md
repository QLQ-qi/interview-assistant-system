# 面试辅助系统 - 完整技术架构

## 1. 系统概述

### 1.1 核心功能流
```
腾讯会议 
    ↓ [实时转写API]
转写文本流
    ↓ [Webhook推送]
本地系统接收
    ↓ [实时存储]
转写记录DB
    ├─→ [知识库检索]
    │    ├─ 标准话术库
    │    ├─ 历史案例库
    │    └─ 岗位资料库
    ↓
本地LLM推理
    ↓ [生成提醒]
面试官实时看板
    ↓
反馈记录 → 知识库沉淀
```

### 1.2 核心特性
- ✅ 实时转写推送处理（毫秒级）
- ✅ 多维知识库检索（向量 + 关键词）
- ✅ 本地LLM推理（隐私、低延迟）
- ✅ 知识库自学习（新录音沉淀）
- ✅ 面试官实时决策支持

## 3. 核心模块设计

### 3.1 Webhook处理模块

**职责**：接收腾讯会议转写推送，完成验签、解析、消息分发

**工作流**：
```
GET /webhook (验证阶段)
  ↓ 计算echo_str = hmac_sha256(check_str)
  ↓ 返回 {echo_str}
  
POST /webhook (推送阶段)
  ↓ 验证签名：verify_signature(body, token, timestamp, nonce)
  ↓ 解析消息类型 (meeting.started / transcription.received / recording.completed)
  ↓ 异步处理
    ├─→ 会议开始事件 → 初始化会话
    ├─→ 转写事件 → 存储 + 知识库查询 + LLM推理
    └─→ 录制完成 → 保存文件路径
```

**关键代码结构**：
```python
# webhook_handler.py
class WebhookHandler:
    def verify_signature(self, token, timestamp, nonce, body):
        """HMAC-SHA256验签"""
        
    def parse_event(self, payload):
        """事件类型识别"""
        
    def handle_transcription_event(self, event):
        """处理转写事件"""
        # 1. 存储转写记录
        # 2. 发起知识库查询 (async)
        # 3. 触发LLM推理 (async)
```

**数据模型**：
```python
class TranscriptionEvent:
    meeting_id: str
    participant_id: str
    text: str              # 转写文本
    timestamp: datetime
    confidence: float      # 转写置信度
    speaker_name: str      # 说话人名字
    sequence_number: int   # 序列号（用于排序）

class WebhookMessage:
    event_type: str
    timestamp: int
    nonce: str
    signature: str
    data: dict
```

## 5. 知识库沉淀流程

### 5.1 从录音到知识库的流程

```
面试录音完成
    ↓
腾讯会议推送录制完成事件
    ↓
下载录音文件
    ↓
音频转录 (ASR) → 完整转写文本
    ↓
文本分段 (切割成话轮)
    ↓
质量评估 (过滤噪音、异常)
    ↓
提取关键信息
├─ 候选人表现评分
├─ 知识点覆盖
├─ 是否通过
└─ 面试官反馈
    ↓
入库知识库
├─ 存储完整转写
├─ 生成向量
├─ 打标签 (岗位/结果/关键词)
└─ 关联元数据
    ↓
用于后续面试辅助
```

## 7. 关键考虑事项

### 7.1 安全性
- Webhook签名验证（HMAC-SHA256）
- API认证 (JWT)
- 敏感数据加密 (面试内容)
- 访问控制 (面试官/HR/系统管理员)

### 7.2 可扩展性
- 异步处理设计
- 消息队列缓冲
- 水平扩展 (多个Worker进程)
- 缓存分层

### 7.3 可靠性
- 重试机制 (指数退避)
- 死信队列处理
- 业务日志记录
- 监控告警 (Prometheus + Grafana)

### 7.4 隐私合规
- 本地部署LLM（不上云）
- 数据本地存储
- 审计日志
- GDPR/隐私法规遵循
